# AGENT

이 문서는 에이전트가 작업을 시작할 때 가장 먼저 보는 얇은 진입점이다.
- 에이전트 모드에서는 모든 비사소한 행동의 첫 게이트를 이 문서로 둔다.
- 즉 새 작업 시작, 단계 전환, 결과 정리, 커밋/푸시 전, 후속 실행 시작 전에는 먼저 이 문서를 다시 읽고 아래 순서로 진입한다.

## 시작 순서

1. 항상 먼저 `docs/README.md`를 읽는다.
2. 바로 이어서 `docs/status.md`를 읽는다.
3. 현재 작업 모드, 현재 목표, 현재 고정 기준을 확인한다.
4. 그 다음에만 관련 모듈 `README`와 필요한 파생 문서를 최소 범위로 읽는다.

## 현재 기본 원칙

- `docs/README.md`가 canonical 문서 허브다.
- `docs/status.md`가 현재 상태의 단일 기준이다.
- `AGENT.md`는 그 둘로 진입시키는 얇은 에이전트용 안내문이다.

## 에이전트 모드 핵심

- 명시적인 중단 지시가 없는 한 이미 시작한 생성, 학습, 변환, 평가, 배포 준비 작업을 끊지 않는다.
- 중간 질문은 기본적으로 상담이나 상태 확인으로 해석하고, 답변 직후 원래 작업 흐름으로 즉시 복귀한다.
- 단계 전환 전, 결과 정리 전, 후속 실행 시작 전에는 다시 `docs/README.md`와 `docs/status.md`를 읽고 기준을 맞춘다.

## 장시간 작업 규칙

- `10분 이상` 걸릴 가능성이 있거나 `여러 단계가 자동으로 이어지는` 작업은 watchdog 대상이다.
- 실전 실행 전에 반드시 watchdog smoke를 먼저 통과시킨다.
- smoke는 최소 아래 셋을 검증한다.
  - 정상 종료
  - 비정상 종료
  - heartbeat 또는 progress가 멈춘 stale 상태
- smoke를 통과하기 전에는 실제 대규모 생성/학습을 걸지 않는다.
- watchdog는 최소 아래 산출물을 남겨야 한다.
  - `pid`
  - `status.local.md`
  - `events.log`
  - `exit code 또는 종료 원인`
  - `다음 단계 실행 여부`

## 보고 규칙

- 현재 실제로 무엇이 돌고 있는지 말할 때는 대화 맥락이 아니라 `ps`, `pid`, 상태 파일, 진행률 파일을 먼저 확인한다.
- 에이전트 모드 답변은 항상 `AI)` prefix를 붙인다.
