## 단계 4. OpenVoice synthetic dataset 생성

### 운영 원칙

- 기본 모드는 `TTS only`다.
- STT 역전사는 **옵션**이다.
- 즉 사용자가 별도로 필터링을 원할 때만 `--enable-stt-filter`를 켠다.
- 대량 생성은 `GPU1` 전용으로 오래 돌리고, 학습/검증은 `GPU0`에서 병행하는 것을 기본으로 한다.

### 실제 생성 스크립트

- script:
  - `repo/tts/tools/openvoice_generate_dataset.py`

### 권장 실행 예시

아래 예시는 현재 canonical text index를 기준으로 **새로운 long run**을 만드는 예시다.

```bash
cd /data2/iena/260318_ondevice-voice-agent
source env/tts_openvoice_v2/bin/activate

export CUDA_VISIBLE_DEVICES=1

python repo/tts/tools/openvoice_generate_dataset.py \
  --corpus-tsv /data2/iena/260318_ondevice-voice-agent/results/tts_custom/corpora/260319_1510_tts_텍스트코퍼스_통합_v1/master_union_unique_by_text.tsv \
  --reference-id ko_female_announcer \
  --reference-audio-path /data2/iena/260318_ondevice-voice-agent/results/tts_custom/references/ko_female_announcer/ko_female_announcer.wav \
  --output-dir /data2/iena/260318_ondevice-voice-agent/results/tts_custom/synthetic_dataset/YYMMDD_HHMM_full_tts_only/openvoice_ko_female_announcer_speed_1p1/shard_01 \
  --model-name KR \
  --voice KR \
  --speed 1.1 \
  --device cuda:0 \
  --max-items 999999 \
  --shard-index 0 \
  --shard-count 1
```

주의:

- `CUDA_VISIBLE_DEVICES=1`로 GPU1만 보이게 했기 때문에, 명령 안의 `--device cuda:0`는 물리 GPU1을 의미한다.
- 이번 성공 run에서는 `full_v1_tts_only`, `full_v2_tts_only`, `full_v3_tts_only`가 이런 방식으로 누적됐다.

### 이미 생성된 run

- `full_v2_tts_only/openvoice_ko_female_announcer_speed_1p1`
