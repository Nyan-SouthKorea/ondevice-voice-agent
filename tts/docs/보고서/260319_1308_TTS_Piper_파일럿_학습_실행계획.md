- 사용자는 나중에 이 sample을 직접 듣고, 추천 checkpoint 외의 다른 checkpoint를 선택할 수 있다.

## 왜 바로 full training으로 가지 않는가

- 현재 synthetic 데이터는 계속 GPU1에서 증가 중이다.
- 한국어 custom TTS는 `OpenVoice teacher 품질`, `Piper 한국어 frontend`, `학습/추론 export`가 모두 동시에 검증돼야 한다.
- 따라서 지금은 아래 두 가지를 동시에 만족시키는 것이 더 중요하다.
  - 시간을 아끼기 위해 작은 pilot로 pipeline 리스크를 먼저 줄인다.
  - full dataset generation은 백그라운드로 계속 쌓는다.

즉 지금 전략은:

1. GPU1
   - OpenVoice full generation 계속
2. GPU0
   - 현재 snapshot으로 Piper pilot 학습

## 현재 선택한 학습 방식

### 현재 선택

- `Piper`
- `single-speaker`
- `scratch`
- `quality=medium`
- `sample_rate=22050`
- `batch_size=32`
- `max_epochs=20`
- `GPU0 전용`

### 왜 scratch를 먼저 택했는가

- 현재 목표는 `최종적인 최고 성능`보다 `한국어 synthetic + Piper 학습 파이프라인이 실제로 성립하는지`를 빠르게 검증하는 것이다.
- transfer learning은 장기적으로 다시 볼 수 있지만, 지금은 영어 checkpoint bias와 frontend 차이를 섞지 않고 한국어 synthetic만으로 성립 여부를 먼저 보는 편이 해석이 쉽다.
- 현재 A100 자원이 충분하므로, pilot에서 굳이 transfer 리스크를 먼저 추가하지 않는다.

### 왜 medium인가

- `x-low`는 너무 빠르지만 최종 Jetson 품질 판단과 차이가 커질 수 있다.
- `high`는 pilot 단계에서 시간이 더 많이 들고, 실패 원인 분리가 어려워진다.
- 따라서 현재는 `medium`이 pilot 대표성 대비 시간 효율이 가장 좋다.

training root:

review root:

- `../results/tts_custom/training/260319_1312_Piper_한국어_파일럿_v1/checkpoint_review/`

정책:

- Lightning checkpoint는 `매 epoch` 저장한다.
- 중요한 checkpoint는 별도로 승격한다.
  - `epoch 0`
  - `epoch 1`
  - 이후 `5 epoch`마다
  - 필요 시 마지막 epoch
- 중요한 checkpoint마다 아래를 함께 저장한다.
  - checkpoint 복사본
  - review prompt 5문장 wav
  - `manifest.tsv`
  - sampler 상태 파일

즉 사용자는 나중에:

- 추천 checkpoint를 듣고
- 필요하면 다른 epoch sample도 같이 들어본 뒤
- 직접 다른 checkpoint를 선택할 수 있다.

## 현재 실행 순서

1. synthetic inventory snapshot 기록
2. `2.0시간` pilot dataset snapshot 생성
3. `piper_train.preprocess`
4. `Piper medium scratch pilot`
5. checkpoint sampler가 중요한 epoch를 자동 보관
6. pilot 종료 후 export / benchmark 연결 여부 판단

## 성공 기준

- preprocess가 정상 완료된다.
- `Piper` pilot training이 실제로 epoch를 진행한다.
- 중요한 checkpoint마다 review sample이 자동 생성된다.
- pilot 종료 후 최소 1개 이상 checkpoint를 export / benchmark 후보로 올릴 수 있다.

## 중단 조건

- preprocess 단계에서 한국어 frontend가 구조적으로 막힌다.
- 학습이 진행돼도 review sample 품질이 명백히 무효다.
- export 또는 downstream 추론 경로가 반복적으로 실패한다.

## 후속 확대 조건

- pilot sample 품질이 baseline 대비 들을 가치가 있으면 full training으로 간다.
- full training은 현재 generation이 계속 쌓이는 동안 `최신 deduped snapshot` 기준으로 다시 잡는다.
- 이때는 현재 `v2` 완료분과 이후 `v3` 생성분까지 포함해 `15~30시간+`로 확대한다.
