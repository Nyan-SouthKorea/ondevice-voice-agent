## 한 줄 요약

- 여성 ref 기준 `Piper` 공식 파인튜닝과 Jetson Nano TTS GUI는 이미 성공 경로가 있다.
- 남성 ref `OpenVoice` 전체 생성은 실제로 완료됐다.
- 하지만 generation 완료 뒤 `inventory -> official fine-tune`으로 이어질 supervisor는 실제로 살아 있지 않았고, stale monitor만 남아 있었다.
- 따라서 다음 실행은 `남성 ref inventory -> official fine-tune -> sampler/postprocess -> A100/Nano smoke`를 detached chain으로 다시 붙이는 것이다.

## 현재 판단

- 남성 ref run은 `synthetic generation` 단계까지는 성공했다.
- 실패 지점은 `generation 완료 후 다음 단계 자동 연결`이었다.
- 따라서 다음 단계의 핵심은 모델 품질 이전에, `실제로 이어지는 detached chain`을 다시 세우는 것이다.

## 다음 즉시 실행 순서

1. `inventory_openvoice_synthetic.py`
   - 입력: 남성 ref generation root
   - 출력: `inventory/`
2. `build_piper_training_dataset.py`
   - 입력: `inventory/deduped_rows.tsv`
   - 출력: `dataset_snapshot/`
3. `piper_train.preprocess`
   - 출력: `preprocessed/`
4. `piper_train`
   - 기준: official `lessac medium` checkpoint resume
   - 출력: `train_root/`
5. checkpoint sampler
6. ONNX export + benchmark postprocess
7. A100 smoke
8. Jetson Nano smoke
9. 결과 보고서 갱신

## 실행 원칙

- 다음 단계는 `세션에 붙은 monitor`가 아니라 `실제 detached 실행`으로 건다.
- 각 단계는 최소한 아래를 남긴다.
  - `pid`
  - `phase.local.md`
  - 단계별 `*.time.log`
  - stdout/stderr 로그
- 다음 단계로 넘어갈 때는 다시 `docs/README.md -> docs/status.md`를 읽고 상태를 맞춘다.
