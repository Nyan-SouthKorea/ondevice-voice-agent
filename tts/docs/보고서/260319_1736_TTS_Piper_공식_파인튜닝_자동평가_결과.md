## 해석

- 속도는 거의 유지됐다.
  - 즉 `Piper family`의 경량성은 유지한 채 intelligibility가 올라간 셈이다.
- 품질은 크게 개선됐다.
  - `0.9149 -> 0.1247`은 단순 noise 감소 수준이 아니라, 실제 발화가 recognizable speech로 들어왔다는 뜻에 가깝다.
- 따라서 직전 실패는 `synthetic 데이터만의 절대적 한계`보다는 아래 영향이 더 컸다고 보는 쪽이 맞다.
  - `scratch` 초기화
  - 공식 fine-tune 레시피 미적용
  - 안정된 prior 없이 synthetic-only 한국어 데이터로 바로 수렴시키려던 점

## 남아 있는 약점

- 숫자, 시간, 단위 표현은 여전히 자주 무너진다.
  - 예: `아홉 시 -> 9시`, `칠십이 퍼센트 -> 72`, `일 점 팔 킬로그램 -> 1 8kg`
- 일부 고유명사성 발음과 띄어쓰기 반영은 불안정하다.
- 이번 run은 checkpoint 보관 정책상 최종 checkpoint만 실제로 남아, 중간 epoch 후보를 사람 귀로 비교하는 선택지는 확보하지 못했다.

## 운영 메모

- 첫 번째 benchmark run `20260319_173438`은 같은 checkpoint가 registry에 두 번 들어가 `success_count=40`, `success_rate=2.0`처럼 집계가 꼬였다.
- 이후 `piper_training_postprocess.py`를 checkpoint stem 기준 dedupe하도록 수정하고, 최종 benchmark는 `20260319_173609`로 다시 수행했다.
- `piper_checkpoint_sampler.py`도 최종 checkpoint는 항상 review 대상에 포함되도록 수정했다.
