  - 같은 `medium` 계열이므로, 최종 Jetson runtime 비교 시 아키텍처 family는 유지된다.

## 이번 런에서 기대하는 개선점

- scratch에서는 초반 acoustic prior가 없어서 synthetic-only 한국어 데이터에서 안정적인 speech manifold에 들어가지 못했을 가능성이 있다.
- fine-tune에서는 기존 영어 single-speaker 모델의 안정된 음성 생성 prior를 유지한 채, 한국어 phoneme/text mapping만 덮어쓰는 방향을 기대할 수 있다.
- 따라서 다음 문제가 줄어들 수 있다.
  - pure buzz/whine collapse
  - 극단적 prosody 붕괴
  - 무의미한 발성으로 수렴

## 그래도 남는 리스크

- synthetic-only 데이터의 teacher artifact
- 한국어 frontend와 영어 pretrained prior의 mismatch
- OpenVoice synthetic prosody가 Piper 학습 분포와 완전히 맞지 않을 가능성
- runtime speed는 같은 Piper 계열이라도 voice, text length, export 설정에 따라 기존 `0.4초`와 완전히 같다고 보장할 수 없음

## 실행 순서

1. 최신 synthetic inventory 기준 full snapshot 생성
2. official lessac medium checkpoint 다운로드
3. `piper_train.preprocess`
4. `resume_from_checkpoint`로 fine-tune 시작
5. checkpoint sampler로 중요한 epoch sample 자동 보관
6. 학습 종료 후 ONNX export + 자동 benchmark

## checkpoint 및 review 정책

- epoch마다 checkpoint 저장
- `epoch 0`, `1`, 이후 `5 epoch`마다 중요한 checkpoint로 승격
- 각 checkpoint마다 review prompt wav 저장
- 사용자는 이후 이 sample을 듣고 추천 checkpoint 외 다른 checkpoint를 선택할 수 있다

## 성공 기준

- checkpoint review sample에서 최소한 intelligible 한국어 음성의 징후가 보인다
- 자동 benchmark에서 scratch pilot 대비 CER가 의미 있게 내려간다
- export와 benchmark 파이프라인이 동일하게 닫힌다

## 중단 조건

- 초기 checkpoint sample부터 명백한 buzz collapse만 반복된다
- 학습이 비정상 종료되거나, checkpoint가 일관되게 손상된다
- GPU0 자원 부족 또는 disk pressure가 심각해진다
